# v1.0.9 Preview Redesign Plan

## Overview

macOS Finder 스타일의 인라인 미리보기를 밀러 컬럼에 도입하고, Git 정보를 하단 상태바로 분리하여 일관된 UX를 제공한다. 모든 뷰 모드에서 Space바 Quick Look 팝업을 통일한다.

## Goals

1. 밀러 컬럼에서 파일 선택 시 인라인 미리보기 자동 표시 (Finder 스타일)
2. Git 정보를 미리보기 패널에서 분리 → 하단 상태바 (모든 뷰 모드 통일)
3. Space바 Quick Look을 별도 플로팅 창으로 변경 (모든 뷰 모드 통일)
4. Details/List/Icon 뷰는 기존 사이드 패널 ON/OFF 유지
5. 미리보기 토글 버튼 역할 재정의

## Current State (v1.0.8)

### 미리보기 패널
- 별도 사이드 패널 (280px 고정, 리사이즈 가능, 최소 200px)
- Ctrl+Shift+P 또는 Space로 토글
- 파일: 콘텐츠 미리보기 + 메타데이터 + Git Tier 1 (마지막 커밋)
- 폴더: 아이템 수 + Git Tier 2 대시보드 (브랜치, 변경파일, 커밋)
- 밀러 컬럼 인라인 미리보기 코드 존재하나 비활성 (UpdateInlinePreviewColumn return)

### Quick Look
- Space바 = 사이드 패널 토글 (HandleQuickLook → TogglePreviewPanel)
- 별도 창 아님

### Git 정보
- 미리보기 패널에 통합 (Tier 1: 파일 커밋, Tier 2: 폴더 대시보드)
- 미리보기 패널이 꺼져있으면 Git 정보 접근 불가

## Target State (v1.0.9)

### 밀러 컬럼 뷰
- 파일 선택 → 인라인 미리보기 컬럼 자동 표시 (남은 공간 adaptive, 최소 200px)
- 폴더 선택 → 다음 컬럼에 폴더 내용 (미리보기 없음, Finder 동일)
- 사이드 패널 불필요 (인라인이 대체)
- 설정에서 인라인 미리보기 OFF 가능 (기본값: ON)

### Details / List / Icon 뷰
- 기존 사이드 패널 ON/OFF 유지 (변경 없음)
- Ctrl+Shift+P로 토글

### Quick Look (모든 뷰 통일)
- Space → 별도 플로팅 창 (포커스 안 가져감)
- ↑↓ 키로 파일 리스트 네비게이션 유지, 팝업 내용 실시간 갱신
- Space 다시 누르면 닫기, ESC도 닫기
- 탭 전환 시 자동 닫기
- 파일만 지원, 폴더 미지원

### Git 상태바 (모든 뷰 통일)
- 탐색기 영역 하단에 1줄 상태바
- Git repo 내부일 때만 표시, 외부면 숨김
- 반응형 3단계:
  - 넓음 (500px+): `🔀 main │ 2 modified · 1 staged · 3 untracked`
  - 중간 (350px+): `🔀 main │ 6 changes`
  - 좁음 (~200px): `🔀 main`
- 클릭 → Flyout (위로 열림, 320px 고정폭)
  - 브랜치명 + 상태 요약
  - 최근 커밋 5개
  - 변경 파일 목록 (최대 20개)
- 밖 클릭하면 닫힘 (WinUI Flyout 기본 동작)
- 탭 전환 시 활성 탭 경로 기준 갱신

### 미리보기 토글 버튼 (툴바)
- 밀러 컬럼 활성: 인라인 미리보기 ON/OFF (기본 ON)
- Details/List/Icon 활성: 사이드 패널 ON/OFF (기존 동작)
- Split view: 기존 pane별 독립 동작 유지

## Implementation Plan

### Phase 1: Git 상태바 분리 (기반 작업)

**목표**: Git Tier 2 대시보드를 미리보기 패널에서 분리하여 하단 상태바로 이동

#### 1.1 Git 상태바 UI 추가
- **MainWindow.xaml**: 탐색기 Grid 하단에 상태바 행 추가
  - 1줄 높이 (~28px), SpanBgLayer2Brush 배경
  - 브랜치 아이콘 + 브랜치명 + 변경 요약 텍스트
  - 클릭 시 Flyout 열기
- **Split view**: 각 pane 하단에 독립 상태바

#### 1.2 Git 상태바 ViewModel
- **새 파일**: `ViewModels/GitStatusBarViewModel.cs`
  - ObservableProperties: Branch, StatusSummary, IsVisible, IsExpanded
  - 반응형 텍스트: SizeChanged 이벤트로 3단계 전환
  - GetRepoInfoAsync 재사용 (GitStatusService)

#### 1.3 Git Flyout UI
- **새 파일** 또는 MainWindow.xaml 내 Flyout
  - 320px 고정폭, 위로 열림
  - 브랜치 + 상태 요약 + 최근 커밋 + 변경 파일
  - 기존 PreviewPanelView.xaml의 Git Tier 2 XAML 재활용

#### 1.4 미리보기 패널에서 Git Tier 2 제거
- **PreviewPanelView.xaml**: Lines 322-365 (Git Tier 2 섹션) 제거
- **PreviewPanelViewModel.cs**: Git Tier 2 프로퍼티 제거 (GitBranch, GitStatusSummary, GitRecentCommits, GitChangedFiles, IsGitRepo)
- **Git Tier 1 (파일 커밋)은 유지** → 인라인 미리보기 메타데이터에 포함

#### 1.5 탭/네비게이션 연동
- 탭 전환 시 GitStatusBarViewModel 갱신
- 폴더 네비게이션 시 repo 경계 감지하여 상태바 표시/숨김
- 다국어 지원 (LocalizationService 연동)

**수정 파일**:
- `MainWindow.xaml` — 상태바 UI 추가
- `ViewModels/GitStatusBarViewModel.cs` — 새 파일
- `PreviewPanelView.xaml` — Git Tier 2 섹션 제거
- `PreviewPanelViewModel.cs` — Git Tier 2 프로퍼티/메서드 제거
- `MainWindow.SplitPreviewManager.cs` — 상태바 초기화/갱신 연동
- `Services/LocalizationData.cs` — 상태바 관련 문자열 추가

- 파일 선택 → 인라인 미리보기 컬럼 표시
- 폴더 선택 → 인라인 미리보기 숨김 (다음 컬럼 표시)

#### 2.2 인라인 미리보기 컬럼 UI
- 기존 PreviewPanelView 재활용 또는 경량 버전 생성

- 사이징: 남은 공간 adaptive (최소 200px)
- ScrollViewer로 세로 스크롤 지원

#### 2.3 사이드 패널과 분리
- 밀러 컬럼 뷰: 인라인 미리보기 사용, 사이드 패널 비활성

  - 밀러 → Details: 인라인 숨김, 사이드 패널 복원 (저장된 상태)

- `SettingsService`: `MillerInlinePreviewEnabled` (기본값: true)
- 설정 > Browsing에 토글 추가

#### 2.5 뷰 모드별 미리보기 상태 독립 저장
- 기존: IsLeftPreviewEnabled (뷰 모드 무관)
- 변경: 뷰 모드별 상태 저장
  - `MillerPreviewEnabled`: 인라인 ON/OFF
  - `DetailPreviewEnabled`: 사이드 패널 ON/OFF
  - `ListPreviewEnabled`: 사이드 패널 ON/OFF
  - `IconPreviewEnabled`: 사이드 패널 ON/OFF

**수정 파일**:
- `MainWindow.SplitPreviewManager.cs` — 인라인 미리보기 활성화/개선
- `MainWindow.xaml` — 인라인 미리보기 컬럼 영역

**목표**: Space바로 별도 플로팅 창 Quick Look 구현

## Implementation Order

```
Phase 1: Git 상태바 분리         ← 가장 독립적, 기반 작업
  ↓
Phase 2: 밀러 인라인 미리보기     ← Phase 1 완료 후 (Git 분리 필요)
  ↓
Phase 3: Quick Look 플로팅 창     ← Phase 2와 병렬 가능
  ↓
Phase 4: 토글 버튼 재정의         ← Phase 2 완료 후
```

## Risk & Mitigation

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 인라인 미리보기 성능 (빠른 파일 선택) | 깜빡임, 지연 | 기존 200ms 디바운스 유지, 캐싱 |
| Quick Look 포커스 관리 복잡 | 키보드 네비게이션 깨짐 | WinUI Popup 사용, 포커스 반환 테스트 |
| Split view + 밀러 + Details 조합 | 상태 관리 복잡 | 뷰 모드별 독립 상태 저장 |
| Git 상태바 공간 (좁은 창) | 정보 잘림 | 반응형 3단계 축약 |
| 기존 사용자 습관 변경 | 혼란 | 설정에서 인라인 미리보기 OFF 가능 |
