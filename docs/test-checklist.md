## 사전 준비

### 테스트 환경
- [ ] Windows 11 Pro (최신 업데이트)
- [ ] .NET 8 Runtime 설치 확인
- [ ] 테스트 폴더 구조 생성 (아래 참조)

### 테스트 폴더 구조
```
C:\SpanTest\
├── Empty\                      (빈 폴더)
├── Normal\                     (파일 10개 + 폴더 5개)
│   ├── SubA\
│   │   └── SubAA\
│   │       └── deep.txt
│   ├── SubB\
│   ├── doc.txt (1KB)
│   ├── image.png (500KB)
│   ├── video.mp4 (5MB)
│   ├── music.mp3 (3MB)
│   ├── data.json (2KB)
│   ├── readme.md (1KB)
│   ├── script.py (500B)
│   └── archive.zip (1MB)
├── LargeDir\                   (파일 1000개 생성)
├── Images\                     (jpg/png/gif/webp 각 10개)
├── DeepNest\A\B\C\...\Z\      (26레벨 중첩)
├── SpecialNames\               (특수 문자 파일)
│   ├── 한글파일.txt
│   ├── file with spaces.txt
│   ├── file(1).txt
│   └── .dotfile
├── ReadOnly\                   (읽기 전용 폴더)
├── HiddenFiles\                (숨김 파일 포함)
│   ├── visible.txt
│   └── .hidden.txt (숨김 속성)
└── ForCopy\                    (복사/이동 테스트용)
    ├── source.txt
    └── conflict.txt            (충돌 테스트용)
```

## Round 1: 스모크 테스트 (핵심 경로)

> 앱이 기본적으로 동작하는지 빠르게 확인

### ST-01. 앱 실행 및 홈 화면 [A]
- [ ] 앱 실행 → 크래시 없이 메인 윈도우 표시
- [ ] 홈 탭 표시 (드라이브 목록 보임)
- [ ] 로컬 드라이브 1개 이상 표시
- [ ] 상태 표시줄 텍스트 표시

### ST-02. 기본 네비게이션 [A]
- [ ] C: 드라이브 클릭 → Miller Columns에서 폴더 목록 표시
- [ ] 폴더 클릭 → 다음 컬럼 생성
- [ ] Backspace → 상위 폴더 이동
- [ ] Ctrl+L → 주소 표시줄 포커스, 경로 입력 후 Enter → 이동

### ST-03. 뷰 모드 전환 [A]
- [ ] Ctrl+1 → Miller Columns
- [ ] Ctrl+2 → Details (테이블 형식)
- [ ] Ctrl+3 → List (세로 흐름)
- [ ] Ctrl+4 → Icon (아이콘 그리드)
- [ ] 각 뷰 모드에서 파일/폴더 정상 표시

### ST-04. 기본 파일 작업 [A]
- [ ] 파일 선택 → Ctrl+C → 다른 폴더 → Ctrl+V → 복사 완료
- [ ] F2 → 이름 변경 TextBox → 새 이름 입력 → Enter → 이름 변경
- [ ] Delete → 파일 삭제 (휴지통)
- [ ] Ctrl+Shift+N → 새 폴더 생성

### ST-05. 탭 [A]
- [ ] Ctrl+T → 새 탭 생성 (홈 모드)
- [ ] 탭 클릭 → 전환
- [ ] Ctrl+W → 탭 닫기
- [ ] 마지막 탭 Ctrl+W → 닫히지 않음 (보호)
